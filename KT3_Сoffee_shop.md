# Техническое задание на разработку Telegram-бота для кофейни

## 1. Введение

Telegram-бот для кофейни, чтобы клиенты могли заказывать кофе и еду прямо в телеграме. Бот нужен, чтобы увеличить количество заказов в то время, когда в кофейне мало людей (например, после обеда). Пользоваться ботом смогут все, у кого есть Telegram.

## 2. Цели и задачи

**Цель проекта:**  
Привлечь больше клиентов в непиковое время (с 14:00 до 17:00), чтобы кофейня зарабатывала больше.

**Задачи:**

1. Сделать бота, в котором можно выбрать кофе и еду из меню.
    
2. Добавить оплату через QR-код.
    
3. Сделать так, чтобы можно было выбрать: забрать заказ самому или заказать доставку.
    
4. Настроить уведомления о том, что оплата прошла и заказ готов.
    
5. Упростить работу сотрудников, чтобы они меньше времени тратили на прием заказов по телефону.
    

## 3. Требования к функционалу

### 3.1. Что может делать пользователь

1. Смотреть меню кофейни (напитки, еда, цены).
    
2. Выбирать несколько товаров в один заказ.
    
3. Видеть свою корзину с суммой заказа.
    
4. Выбирать способ получения:
    
    - Заберу сам
        
    - Доставка (нужно написать адрес)
        
5. Оплачивать заказ по QR-коду.
    
6. Получать сообщения от бота:
    
    - Оплата прошла
        
    - Заказ готовится
        
    - Заказ готов
        

### 3.2. Что может делать администратор (владелец кофейни)

1. Видеть все заказы в специальной веб-панели.
    
2. Менять статусы заказов (в работе, готов, выдан).
    
3. Обновлять меню (добавлять или убирать товары, менять цены).
    
4. Смотреть статистику: сколько заказов, какая выручка.
    

## 4. Требования к интерфейсу

1. Бот должен быть простым и понятным.
    
2. Все действия через кнопки, чтобы не приходилось много печатать.
    
3. После каждого действия бот дает подсказку, что делать дальше.
    
4. Админская панель должна быть удобной с компьютера.
    

## 5. Нефункциональные требования

1. Бот должен быстро отвечать (не дольше 2-3 секунд).
    
2. Бот должен работать стабильно, особенно в часы работы кофейни.
    
3. Данные пользователей (номера, адреса) должны быть защищены.
    
4. Должна быть инструкция для админа кофейни, как пользоваться панелью управления.
    

## 6. Интеграции и окружение

1. Бот будет работать на сервере (например, VPS).
    
2. Будет подключен к платежной системе (ЮKassa или другая) для приема оплаты.
    
3. Будет использовать базу данных для хранения заказов и меню.
    
4. Технологии: Python, PostgreSQL, веб-панель на Django.
    

## 7. Ограничения и допущения

1. Бот работает только в Telegram.
    
2. Доставка только в пределах 2 км от кофейни.
    
3. Оплата только по QR-коду, других способов оплаты в первой версии не будет.
    
4. Обновлением меню в боте занимается сотрудник кофейни через админ-панель.
    
5. Кофейня сама занимается доставкой, бот только передает адрес.
    

## 8. Критерии приёмки

Проект считается готовым, когда:

1. Можно выбрать товары из меню, оформить заказ и оплатить по QR-коду.
    
2. Можно выбрать самовывоз или доставку с указанием адреса.
    
3. Админ видит заказы в веб-панели и может менять их статус.
    
4. Проведено тестирование: 10 человек сделали тестовые заказы, все прошло без ошибок.
    
5. Готова инструкция для администратора кофейни.
